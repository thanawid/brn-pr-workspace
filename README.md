# BRN PR Board

กระดานงานประชาสัมพันธ์ของเทศบาลเมืองบางรักน้อย — **วางแผนงานชัดเจน สื่อสารอย่างมืออาชีพ**

## V1.2.0 — แบบแจ้งงานและรอบเพิ่มความน่าใช้

- เพิ่ม “แบบแจ้งงานประชาสัมพันธ์” แบบกรอกง่าย และบันทึกลงปฏิทินทันทีโดยไม่ต้องกดรับงาน
- คงปุ่มเพิ่มงานเองของทีม PR ไว้ตามเดิม
- ปรับใบสรุป A4 ไม่แสดงชื่อบัญชีทั่วไป เช่น “เจ้าหน้าที่ PR 01” และเว้นช่องผู้บันทึกเมื่อไม่มีชื่อจริง
- แก้ Toast สีดำให้ซ่อนสนิทในสถานะปกติ
- ปรับ Footer พร้อมข้อความ “วางแผนงานชัดเจน สื่อสารอย่างมืออาชีพ” และลิขสิทธิ์
- เก็บฟอนต์ไทย ระยะบรรทัด การตัดคำ ความสูงกล่อง และ Responsive ทั้งระบบ
- เพิ่มภาพประกอบและสีสันแบบแตะเฉพาะจุด โดยไม่เปลี่ยน workflow หลัก

## V1.1.2 — ใบสรุปการปฏิบัติงานประชาสัมพันธ์

- เปลี่ยนหน้าพิมพ์เดิมเป็นเอกสารสรุป A4 หนึ่งหน้า
- ใส่โลโก้งานประชาสัมพันธ์และหัวข้อเทศบาลเมืองบางรักน้อย
- แสดงชื่องาน วัน เวลา สถานที่ กอง/สำนัก ผู้ประสานงาน และสถานะ
- เพิ่มรายการลักษณะงาน เช่น ถ่ายภาพ ถ่ายวิดีโอ เป็นพิธีกร ถ่ายทอดสด เขียนข่าว และเผยแพร่ Facebook
- เพิ่มช่องสรุปการปฏิบัติงาน พร้อมดึงข้อมูลจากงานเดิมอัตโนมัติ
- ปุ่มเดียวสามารถพิมพ์หรือเลือก “บันทึกเป็น PDF” จากหน้าพิมพ์ของเบราว์เซอร์

## V1.1.1 — ปรับลำดับงานหน้าแรก

- งานใกล้ถึงเรียงตามวันที่และเวลา จากใกล้ไปไกล
- แสดงหน้าแรกสูงสุด 5 งาน ลดความยาวของหน้า
- แยกงานค้าง/เลยกำหนดเป็นแถบแจ้งเตือน ไม่ปนกับงานกำลังจะมาถึง
- เพิ่มปุ่ม “ดูงานทั้งหมด” และตัวกรอง “งานค้าง/เลยกำหนด”


## สิ่งที่เปลี่ยนใน V1.1.0

- หน้าแรกแสดงงานวันนี้ งานถัดไป และลำดับงานที่ควรทำต่อ
- สรุปจำนวนงานวันนี้ งานใน 7 วัน งานข้อมูลไม่ครบ และงานรอเผยแพร่
- เพิ่มสถานะงานตั้งแต่รอข้อมูลจนถึงเสร็จสิ้น
- เพิ่มคะแนนความพร้อมและแจ้งเตือนข้อมูลที่ขาด
- เพิ่มใบงาน PR รายกิจกรรม
- เพิ่มผู้ประสานงาน เบอร์โทร ประธาน/ผู้กล่าวเปิด และผลงานที่ต้องจัดทำ
- เพิ่มแนวทางภาพ คลิป การเตรียมงาน และคอนเทนต์ตามบริบทของงาน
- เพิ่มปุ่มพิมพ์ใบงาน คัดลอกบรีฟสร้างสื่อ และส่งแจ้ง LINE อีกครั้ง
- แยกผลการบันทึกงานออกจากผลการส่ง LINE ป้องกันการกดบันทึกซ้ำ
- คลิกวันที่เพื่อดูบอร์ดงานประจำวัน ไม่เปิดฟอร์มเพิ่มงานทันที
- เพิ่มมุมมองรายการสำหรับมือถือและตัวกรองตามสถานะ
- เปลี่ยน “งานของฉัน” เป็น “งานของทีม” ให้ตรงรูปแบบการทำงานร่วมกัน
- แยกข้อมูลวันสำคัญและวันพระออกมาไว้ใน `calendar-data.js` เพื่ออัปเดตเป็นรายปีได้ง่าย

## โครงข้อมูล `prEvents`

ฟิลด์เดิมยังใช้งานได้ และเพิ่มฟิลด์แบบ optional ดังนี้

- `status`
- `contactName`
- `contactPhone`
- `chairperson`
- `outputs`
- `publicationLinks`
- `prTasks`
- `prSummary`
- `allDay`
- `startTime`
- `endTime`

งานเก่าที่ไม่มี `status` จะถูกกำหนดสถานะเบื้องต้นจากวันที่และลิงก์ผลงานโดยอัตโนมัติ จึงไม่ต้องล้างข้อมูลเดิม

## หลังบ้านที่คงเดิม

- Firebase project `brn-pr-board`
- Firebase Authentication
- Firestore collection `prEvents`
- LINE queue `lineOutbox`
- Document `system/lineGroup`
- Cloud Functions `lineWebhook` และ `sendLineOutbox`
- นโยบายแจ้งเตือนล่วงหน้า 1 วันและเช้าวันงาน

## การทดสอบหน้าตาโดยไม่เข้าสู่ระบบ

เปิด `preview.html` ผ่าน local web server เพื่อดูข้อมูลตัวอย่าง โดยไฟล์นี้ไม่ใช้สำหรับขึ้นระบบจริง

## ข้อควรดูแลรายปี

ข้อมูลวันสำคัญและวันพระในชุดนี้ครอบคลุมปี 2569/2026 ให้เพิ่มข้อมูลปีถัดไปใน `calendar-data.js` ก่อนเริ่มปีใหม่ โดยไม่ต้องแก้โค้ดปฏิทินหลัก
