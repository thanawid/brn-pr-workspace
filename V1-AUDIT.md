# BRN PR Board Audit

## สถานะหลังปรับปรุง
- หน้าใช้งานเน้นปฏิทินงานเป็นหลัก
- ตัวช่วยคิดคอนเทนต์ไม่ถูกใช้เป็นป้ายเด่นบนหน้าเว็บ
- ระบบยังจัดแนวทางจากข้อมูลงานจริง วันสำคัญ และวันพระ
- การเพิ่มงานยังเขียนเข้า `prEvents`
- การแจ้ง LINE ยังเข้าคิวผ่าน `lineOutbox`
- Firebase project และ Cloud Functions เดิมยังคงอยู่

## จุดที่ต้องระวัง
- อย่าเปลี่ยน schema งานโดยไม่ตรวจ Cloud Functions และ Firestore rules
- อย่าเพิ่มระบบผู้รับผิดชอบรายคน เพราะโจทย์ปัจจุบันใช้เจ้าของงานระดับกอง/สำนัก
- อย่าทำให้หน้าคำแนะนำเด่นกว่าปฏิทิน
- อย่าโชว์เลขเวอร์ชันบนหน้าเว็บ

## แนวทางต่อยอด
- เพิ่มตัวกรองตามประเภทงานให้ละเอียดขึ้นได้
- เพิ่มมุมมองรายสัปดาห์ได้ แต่ต้องไม่ลดความสำคัญของปฏิทินเดือน
- เพิ่ม template ข้อความโพสต์ได้ โดยให้เรียกจากงานที่เลือกในปฏิทินก่อนเสมอ
