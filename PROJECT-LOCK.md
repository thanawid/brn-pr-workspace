# BRN PR Board — PROJECT LOCK

สถานะ: ข้อกำหนดหลักที่ต้องยึดถือทุกเวอร์ชัน

## นิยามระบบ
BRN PR Board = เครื่องมือบริหารงานประชาสัมพันธ์

## หลักการทำงานจากฐาน V1.6
1. ผู้ใช้เพิ่มงานลงปฏิทินด้วยฟอร์มเดิม
2. ระบุ “เจ้าของงาน” ในระดับกอง/สำนัก เช่น กองสวัสดิการสังคม
3. เมื่อบันทึกงาน ให้แจ้งทีมผ่าน LINE ตามระบบเดิม
4. AI อ่านข้อมูลจาก:
   - ชื่องาน
   - รายละเอียด
   - ประเภทงาน
   - วันสำคัญ
   - วันพระ
5. AI ช่วยแนะนำ:
   - วันนี้ต้องเตรียมอะไร
   - ควรถ่ายภาพแบบไหน
   - ควรเก็บคลิปอะไร
   - คิดคอนเทนต์อะไรได้บ้าง
   - เดือนนี้มีประเด็นอะไรควรทำ
6. ปฏิทินไทยและวันสำคัญอัปเดตอัตโนมัติ
7. แสดง Avatar เล็ก ๆ ของผู้ใช้ที่ล็อกอินในช่องผู้ใช้เดิม ทั้ง Thanawid และ PR01–PR05

## ขอบเขตการแก้ไข
- รักษาฟอร์มเดิมทั้งหมด
- ทำ AI ให้ใช้ข้อมูลจากงานจริง
- ปรับหน้า “แนวทางการทำงาน” ให้เป็นคำแนะนำตามงาน ไม่ใช่คู่มือยาวอย่างเดียว
- เติมรูป Avatar ให้บัญชี PR01–PR05
- ไม่แตะ Firebase, Firestore, Cloud Functions, LINE, Eventarc
- ไม่เพิ่มระบบแบ่งคนถ่าย คนเขียน คนไลฟ์
- ไม่รื้อหน้า
- ไม่เริ่มใหม่

## กติกาเวอร์ชัน
- V1.6 เป็นฐานตั้งต้น ณ วันที่ล็อกเอกสารนี้
- เมื่อมีการแก้ไขและผู้ใช้ยืนยันเวอร์ชันใหม่ ให้เวอร์ชันล่าสุดที่ยืนยันแล้วกลายเป็นฐานใหม่ทันที
- ทุกเวอร์ชันใหม่ต้องต่อยอดจากฐานล่าสุดที่ยืนยัน
- ห้ามย้อนกลับไปใช้ฐานเก่า
- ห้ามลบฟังก์ชันที่ใช้งานได้เดิม เว้นแต่ผู้ใช้สั่งชัดเจน
- ทุกครั้งก่อนแก้ ต้องสรุป:
  1. ฐานเวอร์ชันที่ใช้อยู่
  2. สิ่งที่จะเพิ่มหรือแก้
  3. สิ่งที่คงเดิม
  4. หลังบ้านที่ห้ามแตะ
