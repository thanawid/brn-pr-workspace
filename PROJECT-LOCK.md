# BRN PR Board Project Lock

เอกสารนี้ล็อกทิศทางของระบบหลังปรับโครงสร้างถึง V1.2.14

## แกนหลัก

- งานนำ ปฏิทินนำ ระบบสนับสนุนอยู่เบื้องหลัง
- ผู้ใช้ต้องเห็นงานถัดไป สถานะ ความพร้อม และสิ่งที่ต้องทำก่อนเห็นคำว่า AI
- ไม่ทำหน้า AI Chat เป็นศูนย์กลาง
- ระบบช่วยคิดห้ามสร้างหรือแก้ข้อเท็จจริง เช่น ชื่อ ตำแหน่ง วัน เวลา ตัวเลข และข้อมูลราชการ

## สิ่งที่ต้องรักษา

- ปี พ.ศ. เดือนและวันภาษาไทย
- วันพระ วันหยุด และวันสำคัญ
- ปฏิทินรายเดือนและมุมมองรายการ
- เพิ่ม แก้ไข ลบ และซิงก์งานร่วมกัน
- หน้าแรกสรุปงานวันนี้ งานถัดไป งานข้อมูลไม่ครบ และงานรอเผยแพร่
- สถานะงานและคะแนนความพร้อม
- ใบงาน PR รายกิจกรรม
- แนวทางการเตรียมงาน ภาพ คลิป และคอนเทนต์ตามข้อมูลงาน
- รองรับมือถือ แท็บเล็ต โน้ตบุ๊ก และจอใหญ่
- ไม่มีมาสคอต
- ไม่มีเลขเวอร์ชันแสดงบนหน้าเว็บ

## พฤติกรรมที่ล็อก

- คลิกวันที่: เปิดบอร์ดงานประจำวัน
- เพิ่มงาน: ใช้ปุ่ม “เพิ่มงาน” หรือปุ่มในบอร์ดประจำวัน
- คลิกงาน: เปิดใบงาน PR
- ระบบแนะนำ: คำนวณอัตโนมัติเมื่อเลือกหรือแก้ไขงาน ไม่ต้องกดให้ AI ทำงาน
- การบันทึก Firestore และการแจ้ง LINE ต้องรายงานผลแยกกัน

## ฟังก์ชันที่เพิ่มใน V1.2.0 และต้องรักษา

- ทีม PR ยังเพิ่มงานเองได้ตามเดิม
- แบบแจ้งงานเป็นช่องทางเสริม กรอกแล้วลงปฏิทินทันที ไม่มีขั้นตอนกดรับหรืออนุมัติ
- ใบสรุป A4 ต้องไม่แสดงชื่อบัญชีทั่วไปแทนชื่อบุคคลจริง
- Footer ใช้ข้อความ “วางแผนงานชัดเจน สื่อสารอย่างมืออาชีพ” พร้อมลิขสิทธิ์
- Toast ต้องซ่อนสนิทเมื่อไม่มีข้อความ
- งานปรับภาพลักษณ์ต้องแตะเฉพาะจุด ไม่เปลี่ยน workflow หลัก


## V1.2.1 — มุมมองหลัก
- หน้า “ภาพรวม” ต้องไม่แสดงปฏิทินหรือรายการยาวซ้ำ
- หน้า “ปฏิทิน” ต้องคงปฏิทินและมุมมองรายการครบตามเดิม
- ภาพและ motion เป็นองค์ประกอบเสริม ห้ามแย่งลำดับความสำคัญจากข้อมูลงาน


## V1.2.2 — ปฏิทินในภาพรวมและแบบแจ้งงานสำหรับพิมพ์
- ปฏิทินอยู่ใต้ Hero/การ์ดสรุป และก่อนงานใกล้ถึง
- ปิดช่องทางแจ้งงานออนไลน์ชั่วคราว แต่ทีม PR ยังเพิ่มงานเองได้
- เพิ่มแบบแจ้งงาน A4 สำหรับพิมพ์ให้กอง/สำนักกรอก
- เว้นช่องผู้บันทึกในใบสรุปเป็นเส้นว่างเสมอ
- ปรับ Hero ให้เรียบ พรีเมียม และใช้ภาพหลักเพียงภาพเดียว


## V1.2.3 — ปฏิทินเป็นหน้าหลักและไอเดียคอนเทนต์

- เมนูหลักต้องเรียง ปฏิทิน → ภาพรวม → งานของทีม
- กดโลโก้ต้องกลับไปที่ปฏิทิน
- งานใกล้ถึงหน้าแรกแสดงสูงสุด 3 งาน และมีปุ่มดูงานทั้งหมด
- การ์ด “เดือนนี้สื่อสารอะไรดี?” ต้องแสดงไอเดีย 3 รายการและมีปุ่มคิดใหม่/ดูทั้งหมด
- ไอเดียใช้ข้อมูลจากปฏิทินและวันสำคัญ แต่ห้ามสร้างหรือเปลี่ยนข้อเท็จจริงราชการเอง
- ใช้โลโก้กล้อง–ปฏิทินสีม่วง–ทองเป็นไอคอนหลักและ favicon


## V1.2.4 — ปฏิทินไทยกำลังดี
- เพิ่มสัญลักษณ์มุมช่องในปฏิทินสำหรับวันสำคัญ เช่น ลูกเสือ ภาษาไทย วันแม่ วันชาติ สิ่งแวดล้อม และวันเทศบาล
- วันพระแสดงเงาพระจาง ๆ แบบ watermark พร้อมข้อความขึ้น/แรม/ค่ำ และยังคงอ่านข้อมูลงานได้ชัด
- ใช้โทนม่วง–ทอง–ครีม เพื่อเติมชีวิตให้ปฏิทินโดยไม่เสียความเป็นระบบงาน


## V1.2.5 — ไอคอนลายเส้นลงสีอ่อนแบบชอล์ก / พาสเทล
- เปลี่ยนไอคอนวันสำคัญเป็นแนวลายเส้นลงสีอ่อน ดูชัดแต่ไม่แข็ง
- วันพระใช้ภาพพระพุทธรูปลายเส้นลงสีอ่อนในมุมล่างขวาแบบเห็นได้ชัดขึ้น และไม่แสดงไอคอนซ้ำที่มุมบน
- วันสำคัญอื่นยังแสดงไอคอนเล็กที่มุมช่อง เพื่อเติมชีวิตให้ปฏิทินโดยไม่เสียความเป็นระบบงาน


## V1.2.6 — ใช้รูปภาพและคุมเดือนไม่ให้หลุดโฟกัส
- เปลี่ยนไอคอนวันสำคัญ/วันพระเป็นรูปภาพเล็กใน assets/day-icons เพื่อให้หน้าตานุ่มและสม่ำเสมอขึ้น
- วันที่อยู่นอกเดือนปัจจุบันต้องไม่เด่นเท่ากับเดือนที่กำลังดู: ลดสี ลดความชัด และไม่แสดงรูปตกแต่ง
- ยังคงหลักการ 1 วัน = 1 รูปหลัก และอ่านข้อมูลงานได้เป็นอันดับแรก


## V1.2.7 — ภาพประกอบจริงขึ้นอีกขั้น
- เปลี่ยนชุด day-icons ให้เป็นภาพประกอบสไตล์นุ่มนวลขึ้น มีสีและน้ำหนักภาพมากกว่าไอคอนเส้น
- เอากรอบป้ายของภาพมุมออก ให้ภาพดูเหมือนองค์ประกอบของปฏิทินมากขึ้น
- รักษากติกาเดิม: วันนอกเดือนต้องจางและไม่แสดงภาพตกแต่ง


## V1.2.14 — รูปภาพประกอบ PNG จริง
- ใช้แนวคิด “รูปภาพประกอบ” สำหรับชั้นตกแต่งปฏิทิน
- รูปภาพทั้งหมดเป็นไฟล์ PNG พื้นหลังโปร่งใสใน assets/day-pictures
- วันนอกเดือนแสดงรูปภาพได้ แต่ต้องลดความชัด สี และขนาดเล็กน้อยกว่าวันในเดือนปัจจุบัน
- หนึ่งวันใช้รูปภาพหลักหนึ่งรูป: วันสำคัญมีลำดับเหนือรูปภาพวันพระ แต่ข้อความข้างขึ้น/ข้างแรมยังแสดงอยู่


## V1.2.14 — รูปภาพประกอบจริง
- ใช้รูปภาพสีน้ำที่มีรายละเอียด ไม่ใช่เส้นสัญลักษณ์หรือสติกเกอร์
- รูปวันพระอยู่ใต้ข้อมูลงานและถูกงานบังได้ตามธรรมชาติ
- วันนอกเดือนยังแสดงรูปภาพได้ แต่ลดความเด่นตามระบบเดิม
- ห้ามแตะ layout ระบบเดือน Firebase/Firestore/LINE และฟังก์ชันเดิม


## V1.2.14 — แยกชั้นรูปวันสำคัญ/วันพระให้ถูกต้อง
- รูปวันสำคัญอยู่ด้านบนของช่องปฏิทิน
- รูปวันพระอยู่ด้านล่างเป็นชั้นรอง และถูกงานบังได้
- ถ้าวันเดียวกันมีทั้งวันสำคัญและวันพระ ต้องแสดงได้พร้อมกัน


## V1.2.14 — ลำดับชั้นภาพในปฏิทิน
- รูปวันสำคัญเป็นพื้นหลังชั้นหลังสุดของช่องวันนั้น
- รูปวันพระอยู่ด้านล่าง เป็นคนละส่วนกับวันสำคัญ
- งานและข้อความอยู่ชั้นบนสุดและบังรูปได้ตามธรรมชาติ
- วันเดียวกันแสดงทั้งวันสำคัญและวันพระได้พร้อมกัน
- รอบนี้ไม่แก้ Hero และส่วนอื่น


## V1.2.14 — รูปวันสำคัญครบในปฏิทิน
- วันสำคัญอยู่พื้นหลังชั้นหลังสุด
- วันพระอยู่ด้านล่างแยกกัน
- งานอยู่ชั้นบนสุดและบังรูปได้
- Hero และส่วนอื่นคงเดิม
